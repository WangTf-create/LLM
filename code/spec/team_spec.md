# 团队 API · 规格（SDD 产物）

> 团队工作区：用户创建团队、邀请成员，待办归属团队。所有端点需登录（Bearer JWT）。

## 数据模型

### Team

| 字段 | 类型 | 约束 |
|---|---|---|
| `id` | int | 自增，只读 |
| `name` | str | **必填**；去除首尾空白后 1–50 字；**全局唯一** |
| `created_by` | int | 创建者 user_id，只读 |
| `created_by_username` | str | 创建者用户名（响应用） |

### TeamMember

| 字段 | 类型 | 约束 |
|---|---|---|
| `team_id` | int | 所属团队 |
| `user_id` | int | 成员用户 |
| `role` | str | `owner` 或 `member`；创建者为 `owner` |

同一 `(team_id, user_id)` 唯一。

## 端点

### 1. 创建团队 `POST /teams`

- 请求体：`{ "name": str }`
- 成功：`201`，返回 Team（含 `id`、`created_by` 为当前用户）；创建者自动写入 TeamMember，`role=owner`
- 失败：
  - 名称为空 / 仅空白 / 超过 50 字 → `422`
  - 名称与已有团队重复 → `409`

### 2. 我的团队列表 `GET /teams`

- 成功：`200`，返回当前用户作为成员的 `Team[]`（按 `id` 升序）

### 3. 团队详情 `GET /teams/{team_id}`

- 成功：`200`，返回 Team
- 失败：
  - 非团队成员 → `403`
  - 团队不存在 → `404`

### 4. 邀请成员 `POST /teams/{team_id}/members`

- 请求体：`{ "username": str }`（被邀请用户的 username）
- 权限：**仅 team owner** 可邀请
- 成功：`201`，返回 `{ "team_id", "user_id", "role": "member" }`
- 失败：
  - 非 owner → `403`
  - 非团队成员（非 owner 场景已覆盖）→ `403`
  - 团队不存在 → `404`
  - 被邀请 username 不存在 → `404`
  - 用户已是成员 → `409`

## 边界（会逐条变成测试）

- 团队名为空 `""` → 422
- 团队名仅空白 `"   "` → 422
- 团队名 51 字 → 422
- 重复团队名 → 409
- 非成员访问 `GET /teams/{id}` → 403
- 不存在的 team_id → 404
- member 角色用户尝试邀请 → 403
- 邀请不存在的 username → 404
- 重复邀请同一用户 → 409

## 不在本次范围

- 移除成员、转让 owner、退出团队
- 团队删除、团队重命名
- 邀请链接 / 邮件邀请
- 组织层级（子团队）
